# 4. Пошаговое описание выполнения работы

## Шаг 1. Создание React-проекта

**Действие.** В терминале перешли в папку проекта (например, `C:\web\lab1`) и выполнили команду создания приложения на Vite с шаблоном React:

```
npm create vite@4 . -- --template react
```

**Пояснение.** Команда `npm create vite@4` создаёт новый проект с помощью генератора Vite версии 4 (совместимой с Node.js 18). Точка (`.`) означает «создать проект в текущей папке». Флаг `--template react` задаёт шаблон с настройками под React (JSX, горячая перезагрузка и т.д.).

**Результат.** В папке появились:
- `index.html` — единственная HTML-страница приложения, внутри неё есть `<div id="root">`, куда React встраивает интерфейс;
- `package.json` — список зависимостей и скриптов (dev, build, preview);
- `vite.config.js` — настройки сборщика Vite;
- папка `src/` с файлами `main.jsx`, `App.jsx`, `App.css`, `index.css` и папкой `assets/`;
- папка `public/` для статических файлов.

## Шаг 2. Установка зависимостей

**Действие.** Выполнили в терминале:

```
npm install
```

**Пояснение.** Команда читает `package.json` и устанавливает в папку `node_modules` все перечисленные зависимости (react, react-dom, vite, плагины и т.д.). Без этого шага приложение не запустится.

**Результат.** Появилась папка `node_modules` со всеми библиотеками. В проекте доступны команды: `npm run dev` (запуск в режиме разработки), `npm run build` (сборка), `npm run preview` (просмотр собранной версии).

## Шаг 3. Установка и настройка Tailwind CSS

**Действие 3.1.** Установили Tailwind CSS и необходимые для него пакеты:

```
npm install -D tailwindcss@3 postcss autoprefixer
```

**Пояснение.** Tailwind — библиотека утилитарных CSS-классов (отступы, цвета, сетка и т.д.). Postcss и autoprefixer нужны для обработки стилей. Флаг `-D` ставит пакеты в devDependencies (только для разработки и сборки).

**Действие 3.2.** Создали в корне проекта файл `tailwind.config.js` со следующим содержимым (в сжатом виде):

- В `content` указаны пути ко всем HTML- и JSX-файлам, чтобы Tailwind «просканировал» классы и оставил только используемые в финальном CSS.
- В `theme.extend` при необходимости добавляют свои цвета или отступы (в нашем проекте оставлено пустое расширение).

**Действие 3.3.** Создали файл `postcss.config.js`, в котором подключены плагины `tailwindcss` и `autoprefixer`. Так Vite при сборке прогоняет CSS через Postcss и Tailwind.

**Действие 3.4.** Открыли `src/index.css` и в начале файла добавили три директивы:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

Затем прописали стили для `body`: отступ сверху под фиксированную шапку (`padding-top: 6rem`), цвет фона страницы (светло-зелёный `#f0fdf4`), минимальная высота окна (`min-height: 100vh`), основной цвет текста (тёмно-зелёный `#14532d`).

**Результат.** Все классы Tailwind (например, `bg-white`, `rounded-lg`, `text-emerald-900`) стали работать в компонентах, страница получила общий фон и отступ.

## Шаг 4. Подготовка данных каталога

**Действие.** Создали папку `src/data/` и в ней файл `products.js`.

**Содержимое.** В файле объявлен и экспортирован массив `products`. Каждый элемент — объект с полями:
- `id` — уникальный номер товара (число от 1 до 12);
- `name` — название товара (строка);
- `price` — цена в рублях (число);
- `category` — категория (строка: «Электроника», «Одежда», «Мебель», «Игрушки»).

**Пояснение.** Массив задаёт весь каталог. Он импортируется в `App.jsx` и используется для отображения списка и для фильтрации по категориям и поиску. Список категорий для вкладок получается из этого же массива: из всех товаров берётся поле `category`, из них формируется множество уникальных значений (например, через `[...new Set(products.map(p => p.category))]`).

**Результат.** В приложении есть 12 товаров в четырёх категориях; при добавлении или изменении товаров достаточно править только этот файл.

## Шаг 6. Реализация компонента Header (шапка)

**Действие.** В `Header.jsx` описан компонент, который принимает два пропса: `cartCount` (число товаров в корзине) и `onOpenCart` (функция открытия корзины).

**Реализация.**
- Шапка закреплена сверху экрана (`fixed top-0 left-0 right-0`), фон белый, снизу граница и тень.
- Внутри контейнера с ограничением по ширине (`max-w-6xl`) и отступами: слева заголовок «Каталог товаров», справа кнопка с иконкой корзины (SVG).
- По клику на кнопку вызывается `onOpenCart()`.
- Если `cartCount > 0`, над иконкой корзины выводится бейдж с числом (если больше 99 — показывается «99+»).

**Результат.** На всех страницах видна единая шапка с названием приложения и кнопкой корзины; при добавлении товаров обновляется счётчик.

## Шаг 7. Реализация компонента SearchBar (поиск)

**Действие.** В `SearchBar.jsx` создан компонент с тремя пропсами: `value` (текущий текст в поле), `onChange` (функция, которую нужно вызвать при вводе, передаётся новый текст), `placeholder` (подсказка в пустом поле, по умолчанию «Поиск по названию...»).

**Реализация.** В разметке один элемент `<input type="text">`. Значение поля задаётся через `value={value}` (контролируемый компонент). При изменении текста вызывается `onChange(e.target.value)`. Стили заданы классами Tailwind: белый фон, зелёная рамка и подсветка при фокусе.

**Результат.** В приложении есть единое поле поиска; родитель (App) хранит строку поиска в состоянии и передаёт её в SearchBar, при вводе состояние обновляется и список товаров фильтруется.

## Шаг 8. Реализация компонента CategoryTabs (вкладки категорий)

**Действие.** В `CategoryTabs.jsx` объявлена константа `ALL_CATEGORIES = 'Все категории'` и компонент с пропсами: `categories` (массив названий категорий), `activeCategory` (текущая выбранная категория), `onSelect` (функция смены категории, принимает название категории).

**Реализация.** Сначала рисуется кнопка «Все категории»; по клику вызывается `onSelect(ALL_CATEGORIES)`. Затем в цикле по массиву `categories` выводятся кнопки для каждой категории (Электроника, Одежда, Мебель, Игрушки). У активной кнопки (когда её название совпадает с `activeCategory`) применяются классы «выделенного» состояния (например, зелёный фон), у остальных — нейтральный вид. По клику вызывается `onSelect(cat)`.

**Результат.** Пользователь может переключать фильтр по категориям; выбранная вкладка визуально выделена; родитель хранит `activeCategory` и фильтрует список товаров по ней.

## Шаг 9. Реализация компонента ProductCard (карточка товара)

**Действие.** В `ProductCard.jsx` создан компонент с пропсами: `product` (объект товара), `onAddToCart` (добавить в корзину), `onSelect` (открыть детальную карточку).

**Реализация.**
- Вся карточка обёрнута в кликабельный блок (`onClick={() => onSelect(product)}`), чтобы по клику в любом месте открывалась детальная карточка. Для доступности добавлены `role="button"`, `tabIndex={0}` и обработка нажатия Enter.
- Сверху блок с изображением: соотношение сторон 4:3. Путь к картинке формируется как `/images/${product.id}.jpg` (файлы лежат в `public/images/`). Если картинка не загружается, в состоянии `imageError` сохраняется `true` и вместо фото показывается SVG-заглушка (иконка «картинка»).
- Ниже блок с текстом: категория (мелким шрифтом), название товара, цена в рублях (форматирование через `toLocaleString('ru-RU')`), кнопка «В корзину».
- На кнопке «В корзину» в обработчике вызывается `e.stopPropagation()`, чтобы клик не считался кликом по карточке и не открывал детальное окно; затем вызывается `onAddToCart(product)`.

**Результат.** В каталоге каждая плитка показывает фото (или заглушку), категорию, название, цену и кнопку «В корзину»; клик по плитке открывает детальный просмотр, клик по кнопке только добавляет товар в корзину.

## Шаг 10. Реализация компонента ProductList (список товаров)

**Действие.** В `ProductList.jsx` создан компонент с пропсами: `products` (массив отфильтрованных товаров), `onAddToCart`, `onSelectProduct` (функция открытия детальной карточки, передаётся в каждую ProductCard как `onSelect`).

**Реализация.** Если `products.length === 0`, выводится сообщение «Ничего не найдено» и подсказка изменить поиск или категорию. Иначе рендерится сетка (`grid`): на маленьких экранах одна колонка, на средних две, на больших три, на очень больших четыре (`grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4`). Для каждого элемента массива `products` создаётся компонент `ProductCard` с ключом `key={product.id}`, передаются `product`, `onAddToCart` и `onSelect={onSelectProduct}`.

**Результат.** На странице отображается адаптивная сетка карточек; при пустом результате поиска или фильтра показывается понятное сообщение.

## Шаг 11. Реализация компонента ProductDetail (детальная карточка товара)

**Действие.** В `ProductDetail.jsx` создан компонент с пропсами: `product` (объект товара или null), `onClose` (закрыть окно), `onAddToCart` (добавить в корзину).

**Реализация.**
- Если `product` равен null, компонент ничего не рендерит.
- Состояние `imageError` сбрасывается при смене товара (через `useEffect` по `product.id`), чтобы при открытии другого товара снова пытаться показать его фото.
- Разметка состоит из двух частей. Первая — полупрозрачный фон на весь экран (`fixed inset-0`); по клику и по нажатию Escape вызывается `onClose`. Вторая — белая карточка по центру экрана с `overflow-hidden` и скруглёнными углами. Внутри карточки: блок с изображением (тот же путь `/images/${product.id}.jpg`; при ошибке загрузки — цветной фон и иконка-заглушка), кнопка «Закрыть» (крестик) в углу, затем категория, название, цена, две кнопки — «В корзину» (добавляет товар и закрывает окно) и «Закрыть». Клик по самой карточке не закрывает её (`stopPropagation`), закрывается только по кнопке или по фону.

**Результат.** При клике на плитку товара открывается модальное окно с крупным фото, полной информацией и возможностью добавить товар в корзину или закрыть окно.

## Шаг 12. Реализация компонента CartSidebar (боковая панель корзины)

**Действие.** В `CartSidebar.jsx` создан компонент с пропсами: `isOpen` (открыта ли панель), `onClose` (закрыть), `cartItems` (массив позиций в корзине — объекты с полями `product` и `quantity`), `onRemove` (удалить позицию по id товара).

**Реализация.**
- Итоговая сумма считается как сумма по всем позициям: `price * quantity`.
- Разметка: затемнённый фон (при `isOpen` видимый и кликабельный, иначе с `pointer-events-none`, чтобы не перехватывать клики); по клику вызывается `onClose`. Справа панель фиксированной ширины (`max-w-sm`), при открытии сдвигается в видимую область (`translate-x-0`), при закрытии — вправо за край экрана (`translate-x-full`) с плавной анимацией.
- В панели: заголовок «Корзина», кнопка закрытия (крестик), прокручиваемая область со списком позиций. Каждая позиция — название товара, строка «цена × количество», кнопка удаления (иконка корзины), по клику вызывается `onRemove(item.product.id)`. Если список пуст, выводится текст «Корзина пуста». Внизу при непустой корзине — блок «Итого: … ₽» с посчитанной суммой.

**Результат.** Корзина открывается по клику на иконку в шапке, в ней отображаются все добавленные товары с количеством и суммой, можно удалять позиции и закрывать панель.

## Шаг 13. Сборка логики в App.jsx

**Действие.** В `App.jsx` подключены все компоненты и данные, объявлено состояние и функции, от которых зависит работа каталога, поиска, категорий, корзины и детальной карточки.

**Состояние (useState):**
- `searchQuery` — строка в поле поиска;
- `activeCategory` — выбранная вкладка категории (по умолчанию «Все категории»);
- `cartOpen` — открыта ли боковая панель корзины;
- `cartItems` — массив позиций в корзине, каждая позиция: `{ product, quantity }`;
- `selectedProduct` — объект товара, для которого открыто детальное окно, или null.

**Вычисляемые значения (useMemo):**
- `filteredProducts` — из полного списка товаров сначала отбираются те, у которых `category === activeCategory` (если выбрана не «Все категории»), затем по полю `name` выполняется фильтрация по строке `searchQuery` (без учёта регистра, через `toLowerCase` и `includes`). Пересчёт выполняется только при изменении `activeCategory` или `searchQuery`.
- `cartCount` — сумма всех `quantity` по элементам `cartItems`; используется для бейджа на иконке корзины.

**Функции:**
- `addToCart(product)` — если товар с таким `id` уже есть в `cartItems`, у соответствующей позиции увеличивается `quantity`, иначе в массив добавляется новая позиция `{ product, quantity: 1 }`.
- `removeFromCart(productId)` — из `cartItems` удаляются все элементы, у которых `item.product.id === productId`.

**Разметка (return):** корневой блок с классом `min-h-screen`; внутри — компонент Header (передаются `cartCount` и `onOpenCart={() => setCartOpen(true)}`), основной контент в `<main>` (CategoryTabs, заголовок «Товары», SearchBar, ProductList с `filteredProducts`, `addToCart` и `onSelectProduct={setSelectedProduct}`), компонент ProductDetail (передаются `selectedProduct`, `onClose={() => setSelectedProduct(null)}`, `onAddToCart`), компонент CartSidebar (передаются `cartOpen`, `onClose`, `cartItems`, `onRemove`).

**Результат.** Вся логика фильтрации, корзины и открытия детальной карточки сосредоточена в одном месте; компоненты только отображают данные и вызывают переданные функции.

## Шаг 14. Подключение изображений товаров

**Действие 14.1.** Создана папка `public/images/`. В ней размещён файл `README.txt` с инструкцией: положить в папку изображения с именами `1.jpg`, `2.jpg`, …, `12.jpg` в соответствии с `id` товара в `src/data/products.js`; при использовании формата PNG в коде нужно заменить расширение `.jpg` на `.png`.

**Действие 14.2.** В компонентах `ProductCard` и `ProductDetail` для отображения фото используется путь вида `/images/${product.id}.jpg`. Файлы из `public/` в Vite отдаются с корня сайта, поэтому такой путь указывает на файл `public/images/1.jpg` и т.д.

**Действие 14.3.** Обработка отсутствующего или не загрузившегося изображения: в ProductCard и ProductDetail заведено состояние (например, `imageError`). У элемента `<img>` указан обработчик `onError`, который устанавливает это состояние в true; при true вместо `<img>` рендерится заглушка (цветной блок и иконка или текст). В ProductDetail при смене товара (`useEffect` по `product.id`) состояние ошибки сбрасывается, чтобы для нового товара снова отображалась попытка загрузки его фото.

**Результат.** На плитках и в детальной карточке отображаются изображения из `public/images/`; если файла нет или загрузка не удалась, показывается заглушка без ошибки в консоли.

## Шаг 15. Стилизация интерфейса

**Действие.** Во всех компонентах использованы классы Tailwind в единой цветовой схеме: светло-зелёный фон страницы, белые блоки (карточки, панели, поля ввода), акценты и текст в оттенках зелёного (emerald). Для кнопок применены скругление, отступы, цвет фона и эффект при наведении (`hover:`). Для сетки карточек заданы отступы между элементами (`gap-4`). Шапка и модальные окна имеют тени и границы для визуального отделения. Адаптивность достигнута за счёт префиксов Tailwind (`sm:`, `lg:`, `xl:`), чтобы на разных экранах менялось число колонок и при необходимости размеры элементов.

**Результат.** Интерфейс выглядит единообразно, читаемо и корректно отображается на разных размерах экрана.

## Шаг 16. Запуск и проверка приложения

**Действие.** В терминале выполнена команда:

```
npm run dev
```

**Пояснение.** Vite запускает сервер разработки (обычно на порту 5173) и открывает приложение по адресу http://localhost:5173/. При сохранении файлов страница автоматически обновляется (hot module replacement).

**Проверено:**
- отображение всех товаров при выборе «Все категории»;
- фильтрация по вкладкам категорий;
- поиск по названию (ввод текста, обновление списка);
